# TensorBoard 图表解读指南

详细说明 TensorBoard 中每种图表的含义和如何解读

---

## 📊 SCALARS（标量曲线）

这是最常用的标签页，显示各种数值随时间的变化。

### 1. Loss 曲线

#### Epoch/train_loss（训练损失）
```
作用：监控模型是否在学习
理想曲线：持续下降，无剧烈震荡
问题曲线：
  - 损失不下降 → 学习率过小或模型容量不足
  - 损失剧烈震荡 → 学习率过大或 Batch Size 过小
  - 损失先降后升 → 过拟合，需要早停或正则化
```

**如何判断训练是否成功？**
- ✅ **良好**：损失稳定下降，最终趋于平稳（如 2.5 → 0.6）
- ❌ **欠拟合**：损失始终很高（> 1.5）
- ⚠️ **过拟合**：训练损失很低，但测试损失很高

#### Epoch/test_loss（测试损失）
```
作用：监控泛化能力
理想曲线：与训练损失趋势一致，略高一些
问题曲线：
  - 测试损失远高于训练 → 过拟合
  - 测试损失不下降 → 模型欠拟合或学习率过小
```

**过拟合判断：**
- 正常：Test Loss ≈ Train Loss + 0.1
- 轻度过拟合：Test Loss ≈ Train Loss + 0.2-0.3
- 严重过拟合：Test Loss ≈ Train Loss + 0.5+

---

### 2. Accuracy 曲线

#### Epoch/train_accuracy（训练准确率）
```
作用：监控训练进度
理想曲线：持续上升，趋于平稳
预期数值（FER-2013）：
  - Epoch 1: ~35-45%
  - Epoch 10: ~55-65%
  - Epoch 30: ~65-75%
  - Epoch 50: ~75-85%
```

#### Epoch/test_accuracy（测试准确率）
```
作用：监控泛化能力
理想曲线：与训练准确率相近
判断标准：
  - 正常：Test Acc ≈ Train Acc - 2~5%
  - 轻度过拟合：Test Acc ≈ Train Acc - 10~15%
  - 严重过拟合：Test Acc ≈ Train Acc - 20%+
```

**何时停止训练？**
- Test Acc 不再提升连续 5-10 个 epoch
- Test Loss 开始上升
- 达到预期准确率（如 75%）

---

### 3. Learning_Rate（学习率曲线）
```
作用：监控学习率调度（余弦退火）
理想曲线：从高到低平滑过渡
典型范围：
  - 初始：3e-4 (0.0003)
  - 中期：1.5e-4 (0.00015)
  - 结束：3e-6 (0.000003)

理解：学习率控制模型参数更新速度
  - 太大：震荡、不稳定
  - 太小：收敛慢、可能陷入局部最优
```

---

### 4. Epoch/Time（每个epoch的时间）
```
作用：监控训练效率
预期数值（RTX4060，批大小32）：
  - LIGHT配置：~120-180秒/epoch
  - 50个epoch总计：~2-3小时
```

## 📊 Batch 级别详细记录

### Batch/train_loss（每个batch的损失）
```
作用：监控训练稳定性
理想情况：
  - 整体呈下降趋势
  - 局部有正常波动
  - 没有突增或异常值
```

**异常情况：**
- 🚨 **突然飙升** → 可能是梯度爆炸
- 🚨 **持续高位** → Batch Size 太小或数据质量问题
- 🚨 **周期性波动** → 数据分布不均匀或批采样问题

---

### Batch/train_accuracy（每个batch的准确率）
```
作用：监控单个batch的学习情况
预期范围：
  - 初期：10-30%
  - 中期：40-70%
  - 后期：70-90%
```

---

### 类别准确率（7种表情）

#### Batch/Class_0_Angry_Accuracy（生气）
```
典型准确率：~80-85%
常见混淆：Fear（恐惧）、Neutral（中性）
注意：愤怒表情容易被误判为恐惧
```

#### Batch/Class_1_Disgust_Accuracy（厌恶）
```
典型准确率：~70-75%（最低，样本少）
常见混淆：Angry、Neutral
注意：厌恶样本通常较少，容易欠拟合
```

#### Batch/Class_2_Fear_Accuracy（恐惧）
```
典型准确率：~75-80%
常见混淆：Angry、Surprise
注意：恐惧和愤怒容易混淆
```

#### Batch/Class_3_Happy_Accuracy（开心）
```
典型准确率：~85-90%（最高）
特点：嘴巴特征明显，容易识别
注意：通常是最容易识别的表情
```

#### Batch/Class_4_Sad_Accuracy（悲伤）
```
典型准确率：~75-80%
常见混淆：Neutral、Fear
注意：悲伤和中性有时难以区分
```

#### Batch/Class_5_Surprise_Accuracy（惊讶）
```
典型准确率：~80-85%
常见混淆：Happy、Fear
注意：惊讶通常特征明显（大眼睛、张嘴）
```

#### Batch/Class_6_Neutral_Accuracy（中性）
```
典型准确率：~80-85%
常见混淆：Sad、Angry
注意：中性表情容易被误判为微弱的其他表情
```

**类别准确率分析：**
- ✅ **良好**：所有类别 > 70%
- ⚠️ **一般**：某些类别 < 60%
- ❌ **差**：某个类别 < 50%

**问题类别解决方案：**
1. 增加该类别的样本（数据增强）
2. 调整类别权重（损失函数加权）
3. 专门设计该类别的数据增强

---

### 类别预测率（Batch/Class_X_Pred_Rate）
```
作用：监控模型是否过度偏向某些类别
理想情况：每个类别约 14.3%（1/7）
问题情况：
  - 某个类别 > 20% → 模型过度预测该类
  - 某个类别 < 10% → 模型倾向于避免预测该类
```

**过度预测某类的原因：**
- 该类别样本太多
- 类别权重不平衡
- 数据增强偏向该类别

## 📊 HISTOGRAMS（直方图）

### 损失分布图

#### Epoch/train_loss_distribution
```
作用：查看损失值的分布情况
理想分布：
  - 初期：集中在大值（1.5-2.5）
  - 中期：分布变宽（0.5-2.0）
  - 后期：集中在小值（0.3-0.8）
```

**分布形状解读：**
- 📉 **左偏分布**（大部分小值）→ 训练良好
- 📈 **右偏分布**（大部分大值）→ 模型未学习
- 🌉 **多峰分布**→ 某些样本特别困难或容易

---

#### Epoch/test_loss_distribution
```
作用：查看测试集的损失分布
关注点：
  - 与训练分布的差异
  - 是否有异常高的损失值
```

---

### 权重分布图

#### Weights/patch_embed.proj.weight
```
作用：查看 Patch Embedding 层的权重分布
理想情况：近似正态分布（高斯分布）
问题情况：
  - 梯度消失 → 权重分布集中在0附近
  - 梯度爆炸 → 权重分布范围异常大
```

**梯度问题诊断：**
- **正常**：标准差适中（如 0.01-0.1）
- **消失**：标准差过小（< 0.001）
- **爆炸**：标准差过大（> 1.0）

---

### 梯度分布图

#### Gradients/encoder.layers.*.weight
```
作用：监控梯度健康状况
理想情况：梯度大小适中，分布平滑
问题情况：
  - 梯度为0 → 参数未更新（可能死神经元）
  - 梯度过大 → 参数更新剧烈
  - 梯度震荡 → 训练不稳定
```

**梯度范数（Batch/Gradient_Norm）：**
- 正常：0.1-10
- 消失：< 0.01
- 爆炸：> 100

## 🖼️ IMAGES（图像）

### 混淆矩阵

**解读方法：**
```
    Angry Disgust Fear Happy Sad Surprise Neutral
Angry   500      10     30     20    40     50       100
Disgust  15      200     25     10    20     10        20
Fear     40      15     400    30    50     60        50
Happy    20      10     20    800    30     20        30
Sad      50      20     60     40   500     60        60
Surprise 30      10     50     30    40     600       50
Neutral 100      20     50     30    60     40      800
```

**对角线（正确分类）：**
- 数值越大越好（如 800）
- 对角线亮色表示正确分类多

**非对角线（错误分类）：**
- 数值越小越好
- 深色表示错误少
- 高数值表示混淆严重

**常见混淆模式：**

1. **Angry ↔ Fear**
   - 原因：两者都有皱眉、紧张的相似特征
   - 解决：增强眉毛区域的特征提取

2. **Sad ↔ Neutral**
   - 原因：中性表情和轻微悲伤相似
   - 解决：增强嘴巴区域的特征

3. **Happy ↔ Surprise**
   - 原因：都有张嘴的特征
   - 解决：区分眼睛状态（眯眼 vs 圆眼）

**评估标准：**
- ✅ 优秀：对角线占 > 90%
- ✅ 良好：对角线占 > 80%
- ⚠️ 一般：对角线占 70-80%
- ❌ 差：对角线 < 70%

### Final/Classification_Report（最终分类报告）

**详细指标：**

#### Precision（精确率）
```
定义：预测为某类的样本中，真正属于该类的比例
意义：模型预测的可信度
目标：每个类别 > 75%
```

#### Recall（召回率）
```
定义：真正属于某类的样本中，被正确识别的比例
意义：模型能否找到所有该类样本
目标：每个类别 > 70%
```

#### F1-score（F1分数）
```
定义：精确率和召回率的调和平均
意义：综合性能指标
目标：加权平均 > 80%
```

#### Support（支持度）
```
定义：测试集中每个类别的样本数
用途：检查类别平衡性
理想情况：所有类别样本数相近
```

**优秀模型标准：**
- Accuracy > 75%
- Weighted F1 > 80%
- 所有类别 F1 > 70%
- Macro F1 ≈ Weighted F1（无严重类别偏差）

## 🎯 快速诊断指南

### 问题1：准确率不提升
**检查：**
1. ✅ Loss 是否在下降？（如果否 → 学习率问题）
2. ✅ Batch 级别的准确率是否在上升？
3. ✅ 梯度范数是否正常？（0.1-10）
4. ✅ 权重分布是否健康？

### 问题2：过拟合
**症状：**
- Train Accuracy > 90%
- Test Accuracy < 70%
- Test Loss 上升

**解决方案：**
1. 增加数据增强
2. 增加 Weight Decay（如 0.05 → 0.1）
3. 早停（Best Acc 10个epoch不提升则停止）

### 问题3：欠拟合
**症状：**
- Train Accuracy < 50%
- Test Accuracy < 45%
- Loss 停滞在 > 1.0

**解决方案：**
1. 增大模型（LIGHT → BASE）
2. 增加训练轮数
3. 提高学习率
4. 检查数据质量

### 问题4：某类别识别差
**检查：**
1. 该类别的准确率（Batch/Class_X_Accuracy）
2. 混淆矩阵中的错误类型
3. 数据集中该类别的样本数（Support）

**解决方案：**
1. 数据增强：针对性增强该类
2. 类别权重：加权损失
3. 特征工程：提取该类别的关键特征

## 📊 图表对比示例

### 正常训练曲线

```
Loss: 2.5 → 1.8 → 1.2 → 0.8 → 0.6 → 0.5
Acc:  15% → 35% → 55% → 70% → 78% → 82%
```

### 过拟合曲线

```
Train Loss: 2.5 → 1.0 → 0.3 → 0.2 → 0.1 → 0.05
Test  Loss: 2.5 → 1.5 → 1.2 → 1.3 → 1.4 → 1.5  ← 不降反升！
Train Acc:  15% → 45% → 70% → 85% → 92% → 95%
Test  Acc:  15% → 40% → 60% → 65% → 63% → 60%   ← 停滞/下降！
```

### 欠拟合曲线

```
Loss: 2.5 → 2.3 → 2.2 → 2.1 → 2.0 → 2.0  ← 停滞
Acc:  15% → 20% → 25% → 28% → 30% → 30%  ← 提升慢
```

## 💡 使用建议

### 训练初期（Epoch 1-10）
**重点查看：**
1. ✅ Train Loss 是否下降
2. ✅ Learning Rate 是否按计划调度
3. ✅ Batch Gradient Norm 是否正常

### 训练中期（Epoch 11-30）
**重点查看：**
1. ✅ Test Accuracy 是否稳步提升
2. ✅ Train vs Test Accuracy 差距是否合理（< 10%）
3. ✅ 混淆矩阵是否在对角线集中

### 训练后期（Epoch 31-50）
**重点查看：**
1. ✅ 是否达到目标准确率
2. ✅ 是否有过拟合迹象
3. ✅ 混淆矩阵整体效果

## 🎓 总结

### 最重要的5个图表
1. **Epoch/test_accuracy** → 判断模型性能
2. **Epoch/train_loss** → 判断训练进度
3. **Confusion_Matrix** → 分析错误类型
4. **Final/Classification_Report** → 最终性能评估
5. **Batch/Gradient_Norm** → 诊断训练稳定性

### 判断训练成功的标准
- ✅ Test Accuracy > 75%（FER-2013）
- ✅ 所有类别准确率 > 70%
- ✅ 混淆矩阵对角线占 > 80%
- ✅ Test Loss 稳定下降
- ✅ 梯度范数在 0.1-10 范围内

### 判断需要调整的信号
- 🚨 Test Loss 上升 → 过拟合
- 🚨 梯度爆炸 → 学习率过大
- 🚨 准确率不提升 → 学习率过小或模型问题
- 🚨 某类别准确率低 → 类别不平衡
